Methodology

Making it all work

Initial Processing


Data was collected from the website of the National Basketball Association (NBA), which collects data on every shot taken over the course of a basketball season (in this case, the 2016-2017 season).

Manipulations included removal of opponent shots from the shot log and the 'reflection' of shot coordinates so that all shots appear to have been taken from the same side of the court. This was necessary for our prediction model to work.

An early shot chart generated in Matplotlib for Dwyane Wade, using the NBA data and a basketball court template.

Precision of our ML model

Machine Learning Model


The machine learning model (Gradient Boosting Regressor) was written with SKLearn and cleaned up with SKLearn encode functions.

When we first analyzed the data there were more than 5 positions on the floor (which were discovered to be the two rookies). We dropped these groups to get more accurate feedback. (This brought our model from 41% up to 61%).

Flask App


We created a flask app and wrote code for the machine learning to talk to the website to output a result based on a given input.
We used a dropdown menu to collect a given input to output the machine learning prediction Flask app.

Code for generating a non-geographical map

Application of Leaflet


Our interface and displays relied heavily on the Leaflet library. Leaflet isn't constrained to maps of the Earth and can function using a simple xy coordinate system.
In this case, the map is overlayed with an image of a basketball court so that we can project data or, alternatively, enter queries into our prediction model.
